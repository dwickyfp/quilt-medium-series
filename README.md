# Quilt Medium Series

A hands-on series about [Quilt](https://github.com/dwickyfp/Quilt) — a local-first, visual ETL / ELT desktop studio that compiles pipelines to SQL and runs them on DuckDB.

Each folder is a **self-contained Quilt workspace** plus the Medium article that walks through it. Open the folder as a workspace in the Quilt desktop app, or run its pipelines headlessly through the Quilt engine.

## Series

| # | Folder | Article | What you build | Infra |
|---|--------|---------|----------------|-------|
| 1 | [`01-meet-quilt`](01-meet-quilt/) | Meet Quilt: Local-First Visual ETL | Your first pipeline — CSV → filter → Parquet | Files only |
| 2 | [`02-taming-dirty-data`](02-taming-dirty-data/) | Taming Dirty Data | Validate, standardize, and quarantine bad rows | Files only |
| 3 | [`03-files-to-warehouse`](03-files-to-warehouse/) | From Files to a Warehouse | Load and upsert into PostgreSQL | Docker (Postgres) |
| 4 | [`04-rfm-segmentation`](04-rfm-segmentation/) | Customer Segmentation with RFM | Join sources, score recency/frequency/monetary | Docker (Postgres) |
| 5 | [`05-churn-prediction-ml`](05-churn-prediction-ml/) | Churn Prediction with In-Engine ML | Train, predict, and score a classifier on the canvas | Docker (Postgres) |

## How each workspace is laid out

```
NN-name/
├── ARTICLE.md            # the Medium article (6-part structure)
├── quilt.json            # workspace metadata (open editor tabs)
├── repository.json       # the project tree (pipelines, folders)
├── pipelines/            # one JSON per pipeline
│   └── *.json
├── data/                 # sample input data
├── docker-compose.yml    # only when the case needs a database
└── README.md             # quickstart for that workspace
```

## Running a pipeline

**In the desktop app:** open the series folder as a workspace, pick a pipeline, press Run.

**Headless (via the Quilt engine / MCP):** point the runner at `pipelines/<name>.json`.

For series that need a database, start it first:

```bash
cd NN-name
docker compose up -d
```
