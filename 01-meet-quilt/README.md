# Series 1 — Meet Quilt

The starter workspace for the first article in the series. It builds the simplest possible real pipeline: read a CSV, keep the paid orders, write Parquet.

## Layout

```
01-meet-quilt/
├── ARTICLE.md                     # the Medium article
├── data/orders.csv                # 200 synthetic orders
├── pipelines/orders_to_parquet.json
├── output/                        # paid_orders.parquet lands here on run
├── quilt.json
└── repository.json
```

## Run it

**Desktop app:** open this folder as a workspace, select `orders_to_parquet`, press Run.

**Headless / MCP:** run `pipelines/orders_to_parquet.json`. It reads `data/orders.csv` (200 rows) and writes `output/paid_orders.parquet` (113 paid rows).

## Verify the output

```bash
duckdb -c "SELECT count(*), count(DISTINCT status) FROM read_parquet('output/paid_orders.parquet');"
-- 113 rows, 1 distinct status ('paid')
```

The sample data is regenerated by the snippet in `ARTICLE.md`; no external dependencies.
